# subsidy-monitor

Мониторинг **субсидированных** авиабилетов с алертом в Telegram, когда места
появляются в продаже. Сейчас покрыты Уральские авиалинии (обе стороны Чита ↔ Москва).

## Как это работает
- Строим обычный deep-link booking-движка Ural:
  `book.uralairlines.ru/?model=<JSON>` с субсидированным тарифом (`subsidyType=3`).
- Headless-браузер (Playwright) открывает страницу и **читает рендер**: лента дат
  показывает по ~9 дней с ценой — `от 6 200 ₽` (субсидия есть), `от 22 972 ₽`
  (раскуплено), `-` (рейса нет). Никаких приватных API и токенов.
- Сканируем окно якорями с шагом 7 дней → месяц за 4-5 загрузок.
- Сравниваем с прошлым проходом (`state.json`) и шлём в Telegram **только новые** даты.

## Установка с нуля (новый компьютер / контрибьютор)
```bash
git clone https://github.com/TripIeZ/subsidy-monitor.git
cd subsidy-monitor
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/python -m playwright install chromium   # скачает headless-браузер (~100 МБ)
cp .env.example .env                               # затем впиши токен и chat_id
```
Свой Telegram-бот: создай у **@BotFather** (`/newbot`), токен и `chat_id`
(через @userinfobot или `python3 get_chat_id.py`) впиши в `.env`.

## Запуск
```bash
cd subsidy-monitor
.venv/bin/python monitor.py --once   # один проход (тест)
.venv/bin/python monitor.py          # вечный цикл, раз в 30 мин (POLL_INTERVAL_SEC)
```
Для постоянной работы Mac должен быть включён и не спать (либо вынести на VPS /
поставить launchd-сервис).

## Типы субсидий (Ural)
Бот умеет следить за разными программами субсидий Ural — тип выбирается кнопкой
при создании слежки и хранится в самой слежке. Каждый тип — отдельная модель
booking-движка (`config.URAL_SUBSIDIES`):
- **age** — «Пенсионеры и молодёжь» (пассажир `pensioner`, `isAgeSubsidizedType`).
- **dfo** — «Жители ДФО» (пассажир `adult`, `isFarEastSubsidizedType`). Актуально
  для Читы: всё Забайкалье в ДВФО, субсидия положена по прописке без возрастного ценза.

Добавить программу (инвалиды, многодетные и т.д.) = дописать одну запись в
`URAL_SUBSIDIES`. Проверить модель вживую: `python verify.py 2026-09-15 dfo`.

## S7 — субсидии только вручную (диплинк)
Субсидию S7 **нельзя** мониторить автоматически, и это проверено вживую (2026-07):
- Публичные captcha-free API S7 (`/api/v1/minprice/calendars` и
  `S7WLocationService/api/v1/prices/calendar`) отдают только **обычный** минимум
  (`fareCategory=="default"`), без признака субсидии. Ни один параметр не
  переключает их на льготный тариф.
- Субсидированная цена появляется только внутри `ibe.s7.ru` за Yandex SmartCaptcha.
  Агрегаторы (Aviasales и пр.) субсидии не отдают; NDC Agent API — по договору
  агентства. Открытого `s7-subsidy-monitor` не существует.

Что S7 реально субсидирует Чита↔Москва — **да** (гос-программа 2026, единый тариф
≈ 9 200 ₽): онлайн для молодёжи <23, пенсионеров и **жителей ДФО** (Чита подходит);
многодетные и инвалиды — только через кассы/агентства.

Поэтому в боте S7 сделан как **ручной хендофф**: команда `/s7` или кнопка
«Субсидии S7» (и кнопка после создания слежки) открывают прелинкованный поиск S7
на нужную дату — там человек выбирает свою категорию и проходит капчу.
`config.S7_COMMERCIAL_SCAN` (по умолчанию `False`) — можно включить алерты по
**обычному** минимуму S7, но они честно помечаются «не субсидия».

## Настройка — config.py
- `ROUTES` — маршруты (добавляются одной строкой; MOW = все аэропорты Москвы).
- `URAL_SUBSIDIES` — типы субсидий Ural (модель booking-движка на каждый тип).
- `MONTHS_AHEAD` — глубина сканирования (по умолчанию 4 мес).
- `POLL_INTERVAL_SEC` — частота проверки (1800 = 30 мин).
- `SUBSIDY_PRICE_MAX` — порог цены, ниже которого тариф считаем субсидированным.
- `HEADLESS` — False, чтобы видеть окно браузера (отладка).

## Файлы
| файл | назначение |
|------|-----------|
| `monitor.py` | главный цикл: скан → diff → алерт |
| `ural.py` | проверка Ural (deep-link + парсинг ленты дат) |
| `notify.py` | отправка в Telegram (без зависимостей) |
| `config.py` | маршруты и настройки |
| `state.py` | состояние между проверками (`state.json`) |
| `get_chat_id.py` | узнать свой Telegram chat_id |
| `verify.py` | дебаг: показать рендер субсидии на конкретную дату |
